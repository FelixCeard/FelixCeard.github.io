Programming language

I created my own esoteric programming language called Eb-maj7.

The goal of this programming language was to be able to write functional code and during the execution, music would have been created.

The syntax is very weird as I have (rather had) the plan of converting the execution code to a MIDI file at execution time which would have enabled the user to listen to their program.

In the end, I (only) programmed an interpreter in python (python 3.10s match statements omg soo good!) which is able to execute the syntax as it should.
Even though the syntax is weird, I find it quite interesting as for example, my programming language has only two variables, which is a direct analogy with pianists, who only have two hands to play.
The picture gives an accurate representation how it feels to program with this language.

You can find the code here: GitHub.

Neural Network framework

Having learned functional programming languages in programming one, I wanted to see the limits of this kind of languages, and I challenged myself to create of framework for the training of neural networks.

Because we used an unpopular programming language (SML), it meant that there was no ground truth, so technically I would be the first one to implement gradient descent in this programming language. Note that everything in this "library" is implemented from scratch, including matrix multiplication etc...

My framework is able to train a neural network of any desired size but only consisting of fully connected layers, Relu and sigmoid activation functions.

You can find the code here: GitHub.

Transformer

For the course High Level Computer Vision, I am doing a project involving training a large transformer from scratch.

The goal of the project is to recreate pictures of objects solely on their sketches. Even though this task is not novel, we used the VQ-GAN architecture to introduce a novel approach to this domain.
The advantage of using a transformer instead of a "classic" GAN is that, in theory, it would be able to not only recreate the images based on their sketches, but also to create the sketches from the images. I believe, that with enough data and a bigger transformer, this transformer could excel in both (sub-)tasks.

For our dataset, we used a dataset of 400 thousand amazon product images that we converted with a second AI to sketches.
I used a pretrained VQ-VAE (as described here) which is able to correctly encode and decode both our images and sketches. I Therefore "just" have to train a Transformer, the architecture I am using has 27 millions parameter.
To train this model, it takes around 7-8 hours per epoch. A detailed version of the training can be found in this report: Live report
Note that this is still work in progress.

Game of Real

I stumbled (somewhat on accident) on a version of Game of Life that is in the continuous domain (this YouTube video may help understand what this is all about).

I saw a project on YouTube where someone applied reinforcement learning on the original Game of Life, in order to discover new patterns.
My goal was to recreate this project, but instead of using a discrete space, I would like to use the continuous domain.

As this is computationally intensive (O(n^2)), I wanted to optimize the simulation to run on my GPU.
In the end, I improved the performance from 30s per frame to around 60-70 frames per seconds.

As always, the code can be found here: Github.

Gravity simulation

I create a simulation for the (famous) "n-body problem", which simply states that the gravity between multiple objects (planets) cannot be computed analytically (or fast, sth. along those line).

This naive approach is an approximation of this problem. Unlike the stated problem, I did not focus on the problem itself, but rather on the artistic style of the simulation, as it is very satisfying to see how these planets behave.

Code: GitHub.
